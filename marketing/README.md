# Маркетинг — модуль отчётности по рекламе

Самостоятельное веб-приложение внутри этого репозитория. Адрес: `.../marketing/`.
Единым дашбордом показывает отчётность по рекламным проектам: расход, переходы,
заявки и производные метрики.

## Полная независимость от других приложений

- Свой код и стили в `marketing/` — ничего не импортируется из `finance/` и `planner/`.
- Своя (будущая) схема в базе — `marketing`. Финансы живут в `public`, планер — в
  `planner`. Данные не пересекаются.
- Сейчас прототип **в базу не ходит вообще**: данные демонстрационные, лежат в
  `src/data.js`. Поэтому задеть финансы или планер физически невозможно.

## Карта файлов

```
marketing/
├─ index.html        — точка входа: importmap (Preact+htm), стили, запуск
└─ src/
   ├─ main.js        — запуск приложения (render <App/>)
   ├─ App.js         — весь экран дашборда + компоненты (Card, Sparkline, KPI)
   ├─ data.js        — данные проектов (демо; форма строки = будущая строка в БД)
   ├─ format.js      — форматирование чисел (₽, %, разряды)
   └─ styles.css     — стили только этого модуля
```

## Метрики (формулы — как в гугл-таблице)

- Цена заявки = Расход ÷ Заявки
- Цена перехода = Расход ÷ Переходы
- Конверсия в заявку = Заявки ÷ Переходы

## План развития

1. **Ежедневный робот 06:00** (GitHub Actions, бесплатно) тянет данные из VK Ads
   (`ads.vk.com`, агентский кабинет) и пишет их в схему `marketing` в Supabase.
2. **Приватные ссылки клиентам** — у каждого клиента своя ссылка без логина, видит
   только свой проект. Прямой доступ к таблицам закрыт (RLS); данные отдаёт функция
   по токену из ссылки.

## Статус

Прототип на демо-данных. База, робот VK и клиентские ссылки — впереди.
