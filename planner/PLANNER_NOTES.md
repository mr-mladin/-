## Стандарт доводки (планка) — применять к КАЖДОМУ новому элементу

Проект на уровне Apple. Новое делаем сразу презентабельно, не «лишь бы работало».
Перед коммитом нового UI прогнать чек-лист:
- **Согласованность**: размеры/отступы/радиусы/шрифты — как у соседних элементов; через
  переменные (`--glass`, `--radius-*`, тени), а не случайные числа.
- **Стиль проекта**: копировать существующие паттерны (капсулы, переключатели, стекло,
  emoji-picker), а не изобретать с нуля.
- **Состояния**: hover / active / selected / disabled — везде, где уместно.
- **Без inline-«тяп-ляп»**: стили в CSS-классах, не длинной строкой в разметке.
- **Мелочи**: выравнивание; тень/контур не обрезаются; тёмная тема; иконка/подпись понятны;
  анимация плавная; единый язык с остальным UI.
- Глаз владельца всё равно нужен (я не вижу результат вживую), но криво-по-умолчанию — нельзя.

## Где что лежит (код переплетён — это важно!)

Области интерфейса **не разделены по файлам**. Почти всё — в двух больших файлах:
- `Planner.js` (~2,5K строк) — вся логика и разметка: сетка дня, «весь день», боковое
  дерево, карточки, жесты (перенос, свайпы, зум), клавиатура, недельный/месячный вид.
- `styles.css` (~1,2K строк) — все стили.
- `store.js` — состояние + CRUD (оптимистичные операции, undo/redo, загрузка).
- `lib.js` — клиент Supabase, сетевой слой, даты, повторения, иконки, `errHint`.
- `components.js` — формы (TaskEditor, AuthForm, модалки), `dbHint`/`authHint`.
- `sw.js` — service worker. `index.html` — boot, importmap, версии ассетов.

Вывод: чат «только карточки» всё равно заденет сетку и панель — они в одном коде.
Делить работу лучше по **темам/задачам**, а не по «областям экрана».

## Ключевые механики (где легко сломать)

### Перенос задачи (drag) — самое хрупкое
- Палец/курсор поднимает задачу → рисуется **плавающая копия** (`.tl-lift-overlay`,
  fixed поверх всего), оригинал в сетке прячется (`.lift-hidden`, `visibility:hidden`,
  `transition:none`).
- При отпускании — **«доезд»** (`.landing`): копия едет к слоту за ~0.2с, затем `commit()`
  (reschedule), затем на **следующем кадре** (`requestAnimationFrame`) — `done:true`:
  оригинал показывается уже на месте, копия исчезает. rAF нужен, чтобы оригинал не
  «доезжал» с анимацией top (иначе откат-назад-потом-вперёд).
- `setLiftDrag` в move — **ПРЯМОЙ, без троттлинга.** Троттлинг (rafThrottle) для переноса
  блока вызывал дёрганье «доезда» при малом переносе: отставание = заметный «откат к
  началу» (при большом переносе терялось на фоне движения — отсюда «дёргается без
  перескока, норм с перескоком»). `rafThrottle` остался только для ресайза/панели/дерева.
- Мобильный путь — `onBlockTouch`, десктоп — `startBlockLift`. Логика дублируется.

### Оптимистичные обновления (store.js)
- `updateTaskOptimistic` применяет правку сразу, в фоне пишет в БД. **НЕ перезаписывает
  state серверным ответом** — при быстрых правках (drag, Shift+стрелки) поздний ответ
  устаревает и кидает задачу назад. Ошибку откатывает на `prev`.
- `create`/insert **не повторяем** при сбое (дубли). update/delete — идемпотентны.

### Сетевой слой (lib.js)
- `fetchWithTimeout`: тайм-аут 20с + **автоповтор идемпотентных** (GET/PATCH/DELETE) при
  abort/сетевом сбое. Уважает signal отмены от supabase. Это лечит «нет связи»/«AbortError»
  при рабочем интернете (одиночные обрывы).

### Service worker (sw.js)
- CDN (esm.sh) — cache-first. Свои файлы — **network-first + кэш-фолбэк + тайм-аут 7с**:
  на плохой сети приложение стартует из кэша, а не висит белым/крутилкой. Свежесть кода
  сохраняется (сеть в приоритете).

### Чекбокс и плотность блоков
- Плотность: `height >= 44 ? "" : >= 24 ? "compact" : "mini"`.
- **Чекбокс ВСЕГДА на уровне названия (у верха, `top:7`), НЕ по центру блока.** Название
  и время тоже прижаты к верху во всех режимах — чекбокс и текст на одной линии, позиция
  не зависит от высоты (не прыгает при зуме).
- В compact/mini контент **уменьшается, чтобы помещаться** (чекбокс 17/12px, шрифт мельче).
  **НЕ использовать `overflow:hidden` на `.tl-event`** — он режет тень и чекбокс.
- Выполненная задача: шарик-чекбокс опускается вниз (`--drop = height - 34`, от верха).

### Зум сетки
- Мобильный — **только вертикальный двупальцевый щипок** (пальцы разнесены по вертикали).
  Десктоп — любой щипок / Ctrl+колесо. Класс `.zooming` на время жеста **снимает blur
  стекла** (иначе тяжело) и отключает доводки внутренних элементов.

### Стекло (матовость)
- Переменные `--glass` (заливка), `--glass-blur` (размытие) — **единые во всех разделах**
  (сетка, «весь день», дерево, чип подзадач, неделя, месяц). Тёмная тема — заливка выше.
- **`backdrop-filter` дорог при ДВИЖЕНИИ.** При свайпе дней карусель двигает 3 панели,
  на каждой — стекло у задач; перерисовка размытия ×3 за кадр → рывки/мелькание (сильнее
  «на холодную», сразу после загрузки). Лечится снятием blur на время жеста: класс
  `.swiping` (привязан к `peek`) на `.planner-grid-scroll` убирает `backdrop-filter` и
  доводки — как `.zooming` для зума. **Правило: любую анимацию/перенос поверх стекла
  сопровождать снятием blur на время движения.**

### Тень блока
- Единая для `sel` / `lifted` / `landing`, мягкая: `0 5px 16px rgba(2,6,23,0.15)`.

### Форма задачи (TaskEditor)
- Десктоп — **поповер-облачко СБОКУ** от задачи (`edPopover`: `.ed-float-back.popover`,
  `.ed-card` position fixed, координаты из JS-useLayoutEffect, фон overlay прозрачный —
  сетку НЕ двигаем, задачу не перекрываем). Мобильный — лист снизу / привязанный оверлей
  (`edAnchorMobile`), в сетку класть нельзя (user-select гасит клавиатуру в PWA).
- **«Жидкое стекло»** (`.ed-card`): полупрозрачная (но НЕ слишком — `--glass-card` ~88%/82%
  surface, владелец просил плотнее) заливка + `backdrop-filter` (`--glass-card-blur`),
  **БЕЗ рамки** (`border:0`) — рамка давала видимую линию у основания хвостика; контур даёт
  мягкая тень + верхний блик (`inset 0 1px 0 --glass-hi`), радиус 22px. Внутренние блоки
  (`.ed-headgroup`, `.ed-datebox`) — «слегка серый» фрост `--glass-panel`; пилюли-контролы
  внутри (`.ed-projdot`, `.ed-datepill`, `.ed-timepill`) на `var(--surface)` — выделяются.
  Живое имя: форма зовёт `onLiveTitle` → `liveEdit` в Planner → карточка в сетке (`tl-title`)
  сразу показывает новое название.
- **Верхняя строка** `.ed-top` (как в Apple): слева ✕ (`.ed-icbtn.ed-cancel`, `Icon.close`),
  по центру иконочный переключатель `.ed-typeseg` Задача(`Icon.checklist`)/Событие(`Icon.calendar`),
  справа ✓ (`.ed-icbtn.ed-save`, акцентный, `Icon.check`). `justify-content:space-between` +
  симметричные ✕/✓ → сегмент ровно по центру. Иконки центрируются строго (`line-height:0` +
  `.ico{display:block}` — убирают baseline-зазор inline-svg). Переключение **анимировано**:
  белая пилюля `.ed-typeseg-ind` скользит `translateX` (класс `.on-event` на сегменте).
  Иконка «Задача» `Icon.checklist` — верхний пункт залитый кружок + белая галочка (чтобы не
  «слипался» на 16px), нижний контурный. Степпер проекта `.ed-projdot` — крупнее.
- Оформление события (полоса слева/фон) — **свёрнуто по умолчанию** в строку «Оформление»
  (`styleOpen`), раскрывается по клику (как «Заметка»/«Подзадачи»). Это НЕ баг-пропажа —
  так задумано, чтобы форма не перегружалась 9 плашками сразу при выборе «Событие».
- **Вид под Apple Календарь**: шапка `.ed-head` = название (`textarea`, переносится,
  `field-sizing:content`, ~19px) + справа пилюля-**степпер** проекта (`.ed-projdot`:
  цветная точка + иконка `Icon.stepper()` вверх-вниз; клик → меню выбора).
- **Дата/время/повтор — одна сгруппированная карточка** `.ed-datebox` (как inset-group в
  iOS): строки `.ed-daterow` со светлым фоном и тонкими разделителями (`border-top`).
  Внутри — белые пилюли-контролы на `var(--surface)`: дата `.ed-datepill` (текст
  `medDate` = «12 июня 2026» + прозрачный `<input type=date>` поверх) и `.ed-timepill`
  (нативный `<input type=time>`). Строки: **Весь день** (iOS-тумблер `.ed-switch`,
  зелёный #34c759; вкл → `start_min=null`, показывает одну строку «Дата»; выкл →
  `changeStart("09:00")`, показывает «Начало/Конец»), **Начало/Конец** (дата+время),
  **Повтор** (`.ed-rep-val` + меню `RECUR_OPTIONS`), **Завершить** (дата «до», когда есть
  повтор). Порядок формы: верхняя строка → **блок «Название+Заметка»** → [Оформление
  события] → карточка даты → Подзадачи → Удалить. Старые `.ed-when*`/`.ed-rep-btn`/
  `.ed-until` удалены.
- **Название + Заметка — один блок** `.ed-headgroup` (как «второе поле» в Apple): строка
  названия (`.ed-head`) со степпером проекта справа, ниже — заметка (`.ed-notes`,
  прозрачная, через `border-top`-разделитель). Фон даёт группа, не само поле.
- **Позиционирование облачка** (десктоп, `edPopover`): по реальному rect блока в сетке.
  Задача в верхней половине экрана → форма СНИЗУ (`tail-top`), в нижней → СВЕРХУ
  (`tail-bottom`), узкий блок-колонка (пересечение) → СБОКУ (`tail-left/right`). Хвостик —
  `::after`: бокс той же **стеклянной заливки, что и карточка** (`--glass-card` +
  `--glass-card-blur`), обрезанный `mask`-ом по форме широкого скруглённого треугольника
  (4 SVG-маски `--tail-up/down/left/right`, 36×16 / 16×36, на `.ed-float-back.popover
  .ed-card`). Базой на ~2px заходит под карточку, **карточка без рамки** ⇒ шва нет. КЛЮЧ к
  «части формы» — **тот же материал, что у карточки** (стекло, не `surface`): подхватывает
  оттенок (тинт) и не выглядит белой нашлёпкой. Видимость в тёмной теме держит плотность
  карточки (`--glass-card` ~88–90% surface). Поповеру нужен `overflow:visible`, иначе хвостик

  треугольник + стеклянный материал (проверено офлайн-растеризацией sharp, см. ниже).
  Иконка `Icon.checklist`: два контурных кружка (как в оригинале), у верхнего галочка внутри;
  толщина линий тоньше через `.ed-typebtn .ico{stroke-width:1.8}`.

### Боковая панель (как Apple Календарь)
- Проекты: вместо эмодзи — **цветной чекбокс видимости** (`.proj-check`, скруглённый
  квадрат в цвет проекта). Отмечен = проект показан в сетке (день/неделя/месяц), снят =
  скрыт. Состояние — `hiddenLists` (Set СКРЫТЫХ id + `"inbox"`), **локально в localStorage**
  (`planner.hiddenLists`), НЕ в БД: это настройка вида, у каждого устройства своя.
- Фильтр применяется через `hideFilter` ко ВСЕМ источникам сетки: `dayItems`, `monthItems`,
  `weekDays`, и панелям карусели (`dayStaticPane`, `dayPeekPane`, `buildWeekDays`,
  `buildMonth`). Добавляешь новый источник задач в сетку — не забудь `.filter(hideFilter)`.
- Клик по чекбоксу — `toggleListVisible` (stopPropagation), клик по строке — раскрытие задач.
- Области = **заголовки групп** (`.area-group-btn`/`.area-group-name`, серый мелкий текст +
  шеврон), как «iCloud»/«Другие». Без folder-иконки и счётчика. Правка области: десктоп —
  правый клик (`setCtx`), телефон — кнопки `.area-actions` (на десктопе `display:none`).
- «Все задачи» убран. «Завершено»/«Корзина» — переключатели (повторный клик → `filter:"all"`).
- **Перетаскивание проектов** (reorder + смена области): десктоп — потянул мышью,
  тач — долгое нажатие (быстрый горизонтальный свайп остаётся для кнопок «изменить/
  удалить»). `onProjPointerDown` решает swipe/drag/click; `startProjDrag` ведёт живую
  копию; `projDropAt` ищет зону `[data-arealists]` (область id или `""`=свободные) +
  индекс по серединам строк, либо `[data-areahead]` (в начало свёрнутой области);
  `persistProjOrder` пишет `sort_order` группе и `area_id` перетащенному. `sort_order`
  локален в группе (filter по `area_id` сохраняет относительный порядок). FLIP — `projRects`.

### Клавиатура (десктоп)
- Cmd/Ctrl+Z / Shift+Z — undo/redo. Delete/Backspace — удалить выделенное.
- **Shift+стрелки** двигают выделенные: ↑/↓ — ±5 мин, ←/→ — соседний день (с переездом
  вида и сохранением отрезка прокрутки через `keepScrollRef`, выделение «переезжает»).
- **Cmd/Ctrl+C** — копировать выделенные (в `clipRef` + системный буфер как
  `{__planner:1,snaps}`). **Cmd/Ctrl+V** — наш JSON → `pasteSnaps` (дубль в `dateRef` день,
  тот же проект/время); внешний текст → форма создания с этим названием. Перетаскивание
  текста извне в сетку → `onExternalDrop` (задача на времени точки сброса). Копирование
  пока только из сетки дня (выделение); из бокового дерева — TODO, если попросят.

### Автоперенос просроченного (`rollOverdue`, store.js)
- Переносит на сегодня в «весь день» только **задачи** (`!t.is_event && !recurrence &&
  !recurrence_parent && !done && !deleted_at && date<today`). **События не трогаем** — у них
  нет отметки «выполнено», стоят на своём времени, пока не двинут вручную.

## Грабли (на чём уже обжигались)

- **TDZ → белый экран.** Не использовать `dayTl`/`allDay` (и др. `const`, объявленные ниже)
  в массиве зависимостей `useEffect`, размещённого выше по файлу. Падает `ReferenceError`
  при рендере. → размещать обработчик ПОСЛЕ объявления нужных переменных.
- **`overflow:hidden` на `.tl-event`** обрезает тень и чекбокс. Не применять.
- **rafThrottle без flush** → доезд от устаревшей позиции (дёрганье переноса).
- **Серверная перезапись оптимистичного** → прыжки при быстрых правках.
- **Сырой текст ошибки** наружу («Load failed», «AbortError») — переводить через
  `errHint`/`authHint`/`dbHint` (всё на русском).
- **`backdrop-filter` + движение карусели = мелькание.** Стекло на нескольких панелях,
  перерисовываемых при свайпе/зуме, роняет кадры. Снимать blur на время жеста
  (`.swiping`/`.zooming`). Проявлялось как «моргание» при свайпе дней после загрузки.

## Процесс

- Правки → коммит в рабочую ветку → merge в `main` → push (выкатка в прод, GitHub Pages).
- При правке `styles.css`/`main.js` — **бампать `?v=` в `index.html`** (иначе старый кэш).
  Под-модули (Planner.js и пр.) подтягивает SW свежими, но `?v` на entry — страховка.
- **Перед выкаткой логики — реальный рендер-тест** (мок зависимостей + вызов `Planner()`
  в Node), а не только `node --check`: синтаксис не ловит TDZ/рантайм → белый экран.
  **Готовый харнесс** (контейнер эфемерный — пересоздать заново): `mkdir /tmp/rtest && cd
  /tmp/rtest && npm init -y && npm i preact@10.22.0 htm@3.1.1 preact-render-to-string@6`,
  package.json `type:module`. Скопировать `Planner.js`; сделать мок `lib.js` (реальный, но
  `supabase=createClient(...)` заменить на `{}` и убрать `fetchWithTimeout`), мок `store.js`
  (`export function useStore(){return MOCK}` с tasks/lists/areas/user/ready, `StoreProvider`),
  мок `components.js` (заглушки форм). `test.mjs`: глобальные моки `window/document/
  localStorage/ResizeObserver/requestAnimationFrame/performance` (НЕ трогать `navigator` —
  read-only в Node22), затем `render(html\`<${App}/>\`)` из `preact-render-to-string`.
  Прогонять день/неделя/месяц + скрытые проекты. Ловит TDZ и рантайм-ошибки рендера.
- **Проверка иконок/форм «вслепую»** (нет браузера в среде): растеризовать SVG в PNG через
  `npm i sharp` и смотреть через Read (он показывает картинки). Так подобраны форма хвостика
